Android File Input Test Cases

Expected Path Cases

1. File

  • Display: File browser
  • Restrictions: no file type
  • File Limit: 1
<input type="file" />

2. File - accept image

  • Display: File browser
  • Restrictions: must be an image file type
  • File Limit: 1
<input type="file" accept="image/*" />

3. File - cature

  • Display: Chooser (File or Camera)
  • Restrictions: no file type
  • File Limit: 1
<input type="file" capture />

4. File - accept image and capture

  • Display: Chooser (File or Camera)
  • Restrictions: must be an image file type
  • File Limit: 1
<input type="file" accept="image/*" capture />

5. File - multiple

  • Display: File browser
  • Restrictions: no file type
  • File Limit: many
<input type="file" multiple />

6. File - accept image and multiple

  • Display: File browser
  • Restrictions: must be an image file type
  • File Limit: many
<input  type="file" accept="image/*" multiple />

7. File - capture & multiple

  • Display: Chooser (File or Camera)
  • Restrictions: no file type
  • File Limit: many
<input type="file" capture multiple />

8. File - accept image, capture & multiple

  • Display: Chooser (File or Camera)
  • Restrictions: must be an image file type
  • File Limit: many
<input type="file" accept="image/*" capture multiple />

Unexpected Path Cases

9. File - accept pdf and capture

If accept type is restrictive to a non-image format and capture flag was provided.

  • Display: Chooser (File or Camera)
  • Restrictions: must be a PDF
  • File Limit: 1
<input type="file" accept=".pdf" capture />
